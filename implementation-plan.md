# Implementation Plan

## 阶段概览

| 阶段 | 内容 | 预计产出 |
|------|------|---------|
| 1 | 项目初始化 | 空壳项目可跑 |
| 2 | 主题系统 + 布局框架 | 亮/暗切换、导航栏、响应式骨架 |
| 3 | 个人简介板块 | Hero 区域完整呈现 |
| 4 | 技能展示板块 | 标签徽章网格 |
| 5 | 项目作品板块 | 项目卡片列表 |
| 6 | 联系方式板块 | 社交链接 + 页脚 |
| 7 | 动效与打磨 | 滚动动画、细节优化 |
| 8 | SEO + 部署 | meta tags、GitHub Pages 上线 |

---

## 阶段 1：项目初始化

**目标**：Vite + React + TypeScript + Tailwind 项目跑起来，页面显示 "Hello World"。

**任务**：
- [ ] `bun create vite` 初始化项目（React + TypeScript 模板）
- [ ] 安装 Tailwind CSS 3 + PostCSS + Autoprefixer，配置 `tailwind.config.js`
- [ ] 安装核心依赖：`react-helmet-async`、`framer-motion`、`lucide-react`
- [ ] 安装 devDependencies：`gh-pages`、`eslint`、`prettier`
- [ ] 清理 Vite 默认模板代码（App.css、默认 SVG、默认计数器等）
- [ ] 在 `tailwind.config.js` 中配置灰色系色板、字体、dark mode（`class` 策略）

**验证**：`bun dev` 启动，页面空白无报错。

---

## 阶段 2：主题系统 + 布局框架

**目标**：顶部导航栏可用，支持亮/暗切换，页面有基础骨架。

**任务**：
- [ ] 实现 `useTheme` 自定义 hook（读取 localStorage，切换 `<html>` 的 dark class）
- [ ] 创建 `ThemeToggle` 按钮组件（亮/暗图标切换）
- [ ] 创建 `Navbar` 组件（固定顶部、透明→毛玻璃背景、导航项锚点跳转）
- [ ] 创建 `Layout` 容器组件（max-width 居中、section 间距）
- [ ] 移动端汉堡菜单（导航项收起到侧滑或下拉）
- [ ] 确认响应式断点：手机优先（default）→ `sm` → `md` → `lg` → `xl`

**验证**：导航栏固定在顶部，点击主题按钮全局切换亮/暗，移动端导航折叠正常。

---

## 阶段 3：个人简介板块（Hero）

**目标**：首屏头像、姓名、标签、自我介绍完整展示。

**任务**：
- [ ] 创建 `Hero` 组件
- [ ] 头像区域：占位图（灰色圆形 + 首字母 "六"），后续替换真实图片
- [ ] 姓名 + 职业标签（`<h1>` + `<span>` 标签徽章样式）
- [ ] 三段自我介绍文字，设置合适的行高和间距
- [ ] 移动端：头像缩小、文字左对齐、间距收紧

**验证**：首屏内容清晰可读，桌面和移动端排版合理。

---

## 阶段 4：技能展示板块（Skills）

**目标**：分类标签徽章网格，展示所有技能。

**任务**：
- [ ] 创建 `Skills` 组件
- [ ] 设计 Badge 组件：圆角矩形、灰色系底色、技能名称、小图标（从 lucide-react 选）
- [ ] 按分类分组展示：后端语言 / 前端 / 框架 / 中间件 / 运维 / 软技能
- [ ] 每类一个标题 + 一行 Badge
- [ ] 移动端：Badge 自动换行，间距缩小

**验证**：6 个分类 + 所有技能项正确渲染，暗/亮模式下对比度合理。

---

## 阶段 5：项目作品板块（Projects）

**目标**：4 张项目卡片展示。

**任务**：
- [ ] 创建 `Projects` 组件 + `ProjectCard` 子组件
- [ ] 卡片内容：项目名称、简介、技术栈标签、链接按钮（GitHub / 演示）
- [ ] 桌面端 2 列网格，移动端单列堆叠
- [ ] 卡片 hover 微效（轻微上浮 + 阴影加深）
- [ ] 暂时使用占位项目数据，后续替换

**验证**：4 张卡片按网格排列，链接可点击（跳转 # 占位符）。

---

## 阶段 6：联系方式板块（Contact）

**目标**：邮箱和社交链接展示，页脚收尾。

**任务**：
- [ ] 创建 `Contact` 组件
- [ ] 邮箱展示（图标 + 文字，可复制？还是纯展示）
- [ ] 社交链接行：GitHub / 邮箱（图标 + 标签，点击跳转）
- [ ] 创建 `Footer` 组件（版权、年份、技术栈标注）
- [ ] 导航栏点击项目联动（点击导航项 → 对应板块 scrollIntoView）

**验证**：所有链接可点击，导航锚点跳转平滑。

---

## 阶段 7：动效与打磨

**目标**：克制、少量的滚动进入动画，细节优化。

**任务**：
- [ ] 使用 framer-motion 的 `useInView` + `motion.div` 为每个板块添加淡入上移动画
- [ ] 动画参数统一：`duration: 0.5`、`ease: "easeOut"`、仅触发一次
- [ ] 导航栏滚动时背景变化（透明 → 半透明毛玻璃）
- [ ] 检查所有交互的 `focus-visible` 样式（键盘可访问性）
- [ ] 检查暗/亮模式下所有文字与背景的对比度
- [ ] 全页面在 320px、768px、1024px、1440px 宽度下走查

**验证**：滚动时各板块依次淡入，动画不卡顿，无闪烁。

---

## 阶段 8：SEO + 部署

**目标**：基础 SEO 到位，推送 GitHub Pages 上线。

**任务**：
- [ ] 配置 `react-helmet-async`，添加 `<title>`、`<meta description>`、OG tags
- [ ] 添加 `favicon` 和 `og-image`（先用占位图）
- [ ] 配置 `vite.config.ts` base 路径（`/me/` 或你的仓库名）
- [ ] 编写 GitHub Actions workflow：`push main → build → deploy to gh-pages`
- [ ] 首次手动部署验证：`bun run build && bun run deploy`
- [ ] 检查线上页面各项功能（主题切换、锚点跳转、响应式）

**验证**：`https://<username>.github.io/<repo>/` 可正常访问，所有功能与本地一致。
